# Version-control fixtures

Fixtures covering re-submission and lifecycle behaviour — updating an
already-stored credential, version tracking, supersession. The outer-VP
envelope (bare JSON-LD vs. VC-JWT) is **incidental** to what these tests
assert; the focus is the lifecycle/temporal axis.

These fixtures do **not** advocate Loire conformance. If a future tightening
requires lifecycle scenarios to run on Loire-shape inputs, fixtures here
should be re-signed as VC-JWT VPs rather than relocated.
